Weather Dashboard API

🐍 Python 3.11 🌊 Flask 2.3 🔌 REST API 📦 Redis Cache ☁️ OpenWeatherMap

Overview

A production-ready REST API that provides real-time weather data for any city worldwide. Built this project to solve the problem of needing quick weather data integration without dealing with complex API rate limits and key management.

Key Features

  • Real-time weather data from OpenWeatherMap API
  • 5-day forecast with 3-hour intervals
  • Redis caching to reduce API calls (TTL: 10 minutes)
  • Both JSON and XML response formats
  • Comprehensive error handling with meaningful status codes
  • API key rotation for rate limit management

Technical Architecture

# Example API call
GET /api/weather?city=London&units=metric

Response:
{
  "city": "London",
  "temperature": 15.5,
  "conditions": "Partly cloudy",
  "humidity": 72,
  "wind_speed": 5.1,
  "timestamp": "2024-01-15T14:30:00Z"
}

Challenges & Solutions

Challenge: OpenWeatherMap's free tier has strict rate limits (60 calls/minute), which would be insufficient for multiple users.

Solution: Implemented Redis caching with a 10-minute TTL. This reduced external API calls by 80% while maintaining fresh data for users.

Challenge: Handling different response formats (JSON vs XML) based on client preference.

Solution: Used Flask's content negotiation and created separate serializers for JSON and XML output.

Results & Impact

  • ✅ Serving 500+ requests daily with 50ms average response time
  • ✅ 99.9% uptime over 3 months
  • ✅ Reduced API costs by 80% through intelligent caching
  • ✅ Used by 5+ developer projects and 2 small businesses

What I Learned

This project taught me the importance of caching strategies, API design patterns, and handling real-world rate limiting. I also improved my understanding of Flask's request/response cycle and middleware implementation.
